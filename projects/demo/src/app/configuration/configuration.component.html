# SASS/SCSS

@angular-bulma includes only a minimal set of sass/scss styling on top of the Bulma framework. Unless otherwise specified per-component or per-element, all styling for your application is based on the Bulma styles you load or overwrite.

For example, if you want to use a Bulmaswatch theme instead of the default Bulma stylings, you can simply import it in your main styles.scss:

        
  @import '~bulmaswatch/lux/variables';
  @import '~bulma';
  @import '~bulmaswatch/lux/overrides';

And your application will be themed with the Lux theme!

# Font Icons

@angular-bulma components default to using Font Awesome glyphs for font icons, but @angular-bulma does not load the Font Awesome fonts automatically. Font Awesome must be loaded separately in your styles.scss or from a CDN:

        
@import '~bulma'
@import '~@fortawesome/fontawesome-free/scss/fontawesome';
@import '~@fortawesome/fontawesome-free/scss/regular';

You'll also need to configure your webpack build for Angular to properly handle loading the font files. We suggest looking at @angular-builders/custom-webpack for this.

Alternatively, you can just load Font Awesome from a CDN:

        
<head>
  ...
  <link
    rel="stylesheet"
    href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.0.0-beta3/css/all.min.css"
    integrity="sha512-Fo3rlrZj/k7ujTnHg4CGR2D7kSs0v4LLanw2qksYuRlEzO+tcaEPQogQ0KaoGN26/zrn20ImR1DfuLWnOo7aBA=="
    crossorigin="anonymous"
    referrerpolicy="no-referrer"
  />
</head>

Styles can also be set for other packs of icon fonts. See the documentation for each component/element to see how to style the icons.
